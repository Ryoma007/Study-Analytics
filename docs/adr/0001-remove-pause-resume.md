# 移除暂停/恢复，计时器只保留开始/停止

后端权威计时架构下，ActiveSession 时长 = 结算点 − startTime（连续单段）。若保留暂停/恢复，后端必须维护分段列表并区分"暂停态"与"运行中无心跳"，使心跳协议和结算逻辑显著复杂化。

决定砍掉暂停/恢复，只保留开始/停止。ActiveSession 退化为单段 `{startTime, lastHeartbeatAt}`，心跳只有"我在"一种消息，结算统一为一条公式。用户"歇一下继续"改为停止存一条、再开始一条，对个人时长统计等价，历史里还能看到真实分段记录。

权衡：放弃了"暂停"按钮的肌肉记忆和"一条记录跨多次休息"的连续性，换取后端模型的纯粹性与消除"暂停态 vs 心跳超时"的边界歧义。对学习/阅读这种自我管理场景，时长宁实勿虚，暂停期间本就不该计入。
